# 回合制战斗小游戏 - 产品需求文档

## 项目概述

### 项目简介

一款经典单人回合制对战 H5 小游戏，玩家操控己方角色与地图上的 AI 敌方角色轮流出手战斗，包含普攻、技能、血量、状态、胜负判定等核心回合制逻辑，界面简洁复古，适配移动端与 PC 端，所有数据、逻辑均在前端本地运行。

### 核心目标

- 实现标准回合制战斗流程，回合交替、行动判定逻辑完整
- 界面适配 H5 多端，交互流畅，动画反馈清晰
- 纯前端实现，无后端、无网络请求，刷新可重新开局
- 功能模块化，代码易扩展，支持后续新增角色、技能、Buff
- 玩家每升级一次，物理攻击属性点、法术攻击属性点、防御属性点、速度属性点、血量属性点+1，且额外获得5点属性用于自定义加点，玩家可根据自己的游戏风格和策略选择不同的属性加点。
- 属性加点成长系数：每点物攻点加成物理伤害1.5，每点法攻点加成法术伤害1.1与法力值1.1，每点速度点加成出手速度1，每点防御点加成防御1.2，每点生命上限点加成生命值10。
- **属性点与属性值区分**：
  - 属性点：玩家通过升级、自由分配获得的点数（如玩家物攻点11）
  - 属性值：属性点×系数得出的最终属性数值
  - 装备基础属性：存属性点（需要乘以系数转为属性值）
  - 装备基础词条：存属性点（需要乘以系数转为属性值）
  - 装备强力词条：存属性值（已经乘过系数，直接相加）
- **属性计算公式**：
  - 物理攻击 = 基础物攻值 + 玩家物攻点×物攻系数 + 装备物攻点×物攻系数 + 装备强力物攻值
  - 法术攻击 = 基础法攻值 + 玩家法攻点×法攻系数 + 装备法攻点×法攻系数 + 装备强力法攻值
  - 防御 = 基础防御值 + 玩家防御点×防御系数 + 装备防御点×防御系数 + 装备坚固防御值
  - 速度 = 基础速度值 + 玩家速度点×速度系数 + 装备速度点×速度系数 + 装备迅捷速度值
  - 生命上限 = 基础生命上限值 + 玩家生命上限点×生命系数 + 装备生命上限点×生命系数 + 装备强壮生命值
  - 法力上限 = 基础法力上限值 + 玩家法攻点×法力系数 + 装备法攻点×法力系数 + 装备浩瀚法力值
  - **注意：法力上限不依赖独立的属性点，而是通过法攻点自动增加**
  - 基础属性值（不依赖属性点）：
    - 基础物攻值：15
    - 基础法攻值：10
    - 基础防御值：5
    - 基础速度值：5
    - 基础生命上限值：100
    - 基础法力上限值：50
  - **暴击率、连击率、最大连击次数**：
    - 这三个属性是玩家和敌人的战斗属性，但**不可自由加点**，**升级也不会自动增加**，仅用于战斗计算和面板显示
    - 玩家默认值：
      - 基础暴击率：5%
      - 基础连击率：0%
      - 基础最大连击次数：1次
    - 敌人随机范围：
      - 暴击率：0% ~ 15%
      - 连击率：0% ~ 20%
      - 最大连击次数：1 ~ 3次
    - 暴击率计算：
      - **实际暴击率 = 玩家暴击率（可被调试面板修改） + 所有装备强力词条暴击率**（受最大暴击率上限限制）
      - 敌人暴击率：随机生成（不受装备加成影响）
      - 暴击伤害 = 基础伤害 × 暴击倍数（暴击倍数可配置）
    - 连击率计算：
      - **实际连击率 = 玩家连击率（可被调试面板修改） + 所有装备强力词条连击率**（受最大连击率上限限制）
      - 敌人连击率：随机生成（不受装备加成影响）
      - 连击时，每次连击的概率会衰减（可配置衰减系数，默认50%）
      - 连击伤害逐次递减（可配置递减系数，默认70%）
      - 最大连击次数受最大连击上限限制（可配置，默认99次）
    - 最大连击次数计算：
      - **实际最大连击次数 = 玩家最大连击次数（可被调试面板修改） + 所有装备强力词条最大连击次数**（受最大连击次数上限限制）
      - 敌人最大连击次数：随机生成（不受装备加成影响）
    - **战斗计算示例**：
      - 玩家暴击率5%，敌人随机暴击率0%-15%
      - 玩家连击率0%（基础），敌人连击率0%-20%
      - 玩家最大连击1次（基础），敌人最大连击1-3次
      - 连击概率衰减：每次连击后，触发下一次连击的概率衰减50%
      - 连击伤害递减：每次连击的伤害为基础伤害的70%
    - **装备词条支持**：
      - 暴击率词条：武器专属，取值范围[1, 30]，系数1
      - 连击率词条：武器专属，取值范围[1, 30]，系数1
      - 最大连击词条：玉佩和项链专属，取值范围[1, 10]，系数1
      - 全属性词条：护胸甲和护腿甲专属，取值范围[1, 装备等级]，生效加成如下：
        - physicalAttack + 词条值
        - magicAttack + 词条值
        - defense + 词条值
        - speed + 词条值
        - maxHp + 词条值
        - maxMp + 词条值
        - critRate + (词条值/装备等级 * 30)，最小为1
        - comboRate + (词条值/装备等级 * 30)，最小为1
        - maxComboCount + floor(词条值/装备等级 * 10)，最小为1
  - **敌人物理攻击和法术攻击**：
    - 敌人区分物理攻击和法术攻击，不同敌人有不同的物理攻击力和法术攻击力
    - 敌人配置包含：physicalAttack（物理攻击力）、magicAttack（法术攻击力）、attackType（攻击类型）
    - 攻击类型：
      - physical：物理型敌人，主要使用物理攻击
      - magic：法术型敌人，主要使用法术攻击
    - 敌人伤害计算：基础伤害 = max(1, 敌人攻击力 - 玩家防御力)
  - **敌人使用技能**：
    - 敌人可以配置可以使用的技能列表（skills 数组）
    - 敌人在战斗中有50%的概率使用技能（如果有技能可用）
    - 敌人的技能伤害 = 技能基础伤害 + 敌人魔法攻击 × 0.5
    - 敌人技能也可以暴击（基于敌人的暴击率）
  - 示例：玩家物攻点11，装备物攻点12，强力物攻值110，物攻系数1.5
    - 实际物理攻击 = 15 + 11×1.5 + 12×1.5 + 110 = 15 + 16.5 + 18 + 110 = 159.5（向下取整为159）
- 装备系统：击败敌人概率获得装备，装备类型包括武器、项链、头冠、玉佩、护胸甲、护腿甲、鞋子、左手环、右手环、腰带，每种装备有独立词条库（至少20种词条）和专属加成。
- 每件装备最多拥有5种词条，最少0种词条，词条从该种装备的词条库随机选择。
- 装备词条数值与装备等级关联：基础词条（物攻/法攻/防御/速度/法力/生命）取值范围为[1, 装备等级+2]。
- 强力词条（强力物理攻击/强力法术攻击/坚固防御/迅捷速度/强壮生命/浩瀚法力）加成属性值而非属性点，取值范围为[1*属性系数, (装备等级+2)*属性系数]。
- 金币系统：击败敌人可获得金币奖励，金币用于购买道具、装备或刷新装备词条。
- 装备等级系统：装备有等级属性，等级范围为[玩家等级-2, 玩家等级+5]，基础属性随等级上升而增加。
- 商店系统：可消耗金币购买道具和装备（无词条，只有基础等级），可出售道具获得金币，可消耗金币刷新装备词条。
- 词条颜色系统：装备的每一条词条根据数值大小显示不同颜色（低数值灰色、中数值蓝色、高数值紫色、极高数值橙色）。

## 功能详细需求

### 页面结构

- 地图上散布着 AI 敌方角色，玩家需要与之战斗
- 玩家角色可在地图上移动，与 AI 敌方角色进行战斗，击败 AI 敌方角色后可获得升级经验和概率获得装备技能书等奖励。
- 非战斗时，玩家可查看当前装备，背包，当前拥有的技能，当前角色的属性（血量、状态、加点属性等）
- 战斗时，
  - 上方：敌方 AI 角色（头像、名称、血条）
  - 下方：己方角色（头像、名称、血条、法力条）
  - 战斗信息区：显示当前回合、当前行动角色、当前行动类型（普攻、技能、状态等）、显示当前行动方提示（玩家回合 / 敌方回合）
  - 日志区：显示战斗中的所有操作记录，包括玩家操作、AI 操作、伤害计算等。
  - 操作按钮区（玩家回合可点击，敌方回合置灰禁用）
    - 普攻按钮
    - 技能按钮（根据角色属性动态显示）
    - 道具按钮（可使用背包中的回血道具，回法力道具）
    - 防御按钮（本回合减伤）
    - 逃跑按钮（点击后结束战斗，返回地图界面）
  - 结算界面（战斗结束时显示）
    - 显示战斗结果（玩家胜利 / 敌方胜利）
    - 显示战斗中获得的经验值、装备、技能书等奖励

### 回合流程

- 默认先手方根据玩家和怪物的速度属性点判断，玩家速度属性点高则先手，否则怪物先手。
- 行动方执行操作 → 结算伤害 / 状态 → 切换行动方
- 循环回合，直到任意一方血量 ≤ 0，战斗结束
- 敌方回合由AI 自动执行，无需玩家操作

### 玩家行动指令规则

- 物理攻击：根据玩家角色的物理攻击属性点，对单个敌方角色造成大量物理伤害。
- 法术攻击：根据玩家角色的法术攻击属性点，消耗玩家角色的法力值，对单个/或多个敌方角色造成法术伤害（根据法术类型不同，对敌方数量不同）。
- 防御：根据玩家角色的防御属性点，对敌方角色的伤害进行额外的减伤。仅当前回合生效，下一回合效果清除。无伤害输出，仅防御自身收到的伤害。

### 敌方行动指令规则

- 敌方使用技能不消耗法力值。
- 敌方使用道具不消耗道具。
- 敌方血量<=30%时候，概率使用回血道具

### 胜负判定

- 玩家血量 ≤ 0 → 战斗失败，跳转结算页
- 敌方血量 ≤ 0 → 战斗胜利，跳转结算页
- 角色阵亡后，停止所有回合行动

### 战斗日志

日志内容示例：
「玩家 使用普通攻击，敌方受到 20 点伤害」
「敌方 释放技能，玩家受到 35 点伤害」
「玩家 进入防御状态，本回合减伤 50%」
「蓝量不足，无法释放技能！」

### 存储

使用本地缓存

### 扩展预留

- 新增多角色选择、多套技能
- 增加 Buff/Debuff 状态（中毒、眩晕、暴击）
- 新增道具系统（回血药、回蓝药）
- 多关卡、难度递增
- 地图等级系统：
  - 地图有等级属性，玩家可在地图界面设置
  - 敌人属性与地图等级相关联：敌人属性 = 基础属性 × (1 + 1.3 × (地图等级 - 1))
  - 敌人技能使用限制：
    - 地图等级前5级，敌人不会使用辅助技能
    - 地图等级前8级，敌人不会使用障碍技能
- 障碍技能系统：
  - 封印术：小概率成功，封印后敌人每回合跳过行动，持续2回合
  - 中毒术：概率成功，每回合扣减血量和法力，持续3回合
  - 冰冻术：大概率成功，每回合跳过行动但免疫伤害，持续2回合
  - 混乱术：小概率成功，每回合随机攻击不分敌我，持续2回合
- 敌人混乱状态行动规则：
  - 混乱状态下，敌人不分敌我随机选择目标（玩家、宠物或友方敌人）
  - 混乱状态下，敌人只使用物理攻击或法术攻击，不能使用辅助技能和障碍技能
  - 混乱状态下，敌人随机选择物理攻击或法术攻击（50%概率）
  - 混乱状态下使用法术攻击时会消耗法力值，但法力不足时仍可使用不扣法力
- 战斗奖励系统：
  - 战斗奖励（金币、装备等级等）与地图等级相关联
