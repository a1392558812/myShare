# 回合制战斗小游戏 - 产品需求文档

## 项目概述

这个demo是一个技术验证demo，尝试一行代码也不写。前期我只梳理了一个游戏的大概的需求，比如:
1. 模式是回合制，玩家能通过个性化的属性点分配来影响战斗
2. 玩家可通过升级获得属性点，也可通过装备来获得加成
3. 玩家分两种攻击模式，普通攻击和技能攻击等
4. ui我让ai随便整的，主要是实现功能，看看ai对人类描述的逻辑的理解是否正确。

后续的一些零碎需求，包括这篇文档，是我让ai根据我提到的零碎需求生成的。不得不感叹ai的确实很牛皮，但也在迭代中发现，ai擅长的领域是从0-1，而不是需求迭代和逻辑微调，可能是我描述有歧义，也可能是ai的token过长导致的ai幻觉等等，总之出现了一系列问题，例如：

1. 生成的代码不应要求
2. 莫名其妙的改/删一些看似关联的代码实则只是同名的函数/变量
3. 莫名其妙的忘记了之前的逻辑需求

等等,刚开始很顺手，越到后面迭代奇怪问题出现频率越多，还是需要人类更细致的提出需求和排查代码逻辑，给ai擦腚😂😂😂

### 项目简介

一款经典单人回合制对战 H5 小游戏，玩家操控己方角色与地图上的 AI 敌方角色轮流出手战斗，包含普攻、技能、血量、状态、胜负判定等核心回合制逻辑，界面简洁复古，适配移动端与 PC 端，所有数据、逻辑均在前端本地运行。

### 核心目标

- 实现标准回合制战斗流程，回合交替、行动判定逻辑完整
- 界面适配 H5 多端，交互流畅，动画反馈清晰
- 纯前端实现，无后端、无网络请求，刷新可重新开局
- 功能模块化，代码易扩展，支持后续新增角色、技能、Buff

## 核心属性系统

### 属性点与属性值区分

- **属性点**：玩家通过升级、自由分配获得的点数
- **属性值**：属性点×系数得出的最终属性数值
- **装备基础属性**：存属性点（需要乘以系数转为属性值）
- **装备基础词条**：存属性点（需要乘以系数转为属性值）
- **装备强力词条**：存属性值（已经乘过系数，直接相加）

### 属性计算公式

| 属性 | 计算公式 |
|------|----------|
| 物理攻击 | 基础物攻值(55) + 玩家物攻点×1.5 + 装备物攻点×1.5 + 装备强力物攻值 |
| 法术攻击 | 基础法攻值(10) + 玩家法攻点×1.1 + 装备法攻点×1.1 + 装备强力法攻值 |
| 防御 | 基础防御值(5) + 玩家防御点×2 + 装备防御点×2 + 装备强力防御值 |
| 速度 | 基础速度值(5) + 玩家速度点×1 + 装备速度点×1 + 装备强力速度值 |
| 生命上限 | 基础生命上限值(350) + 玩家生命上限点×10 + 装备生命上限点×10 + 装备强力生命值 |
| 法力上限 | 基础法力上限值(50) + 玩家法攻点×1.1 + 装备法攻点×1.1 + 装备强力法力值 |

> **注意**：法力上限不依赖独立的属性点，而是通过法攻点自动增加

### 战斗属性（不可加点）

| 属性 | 玩家基础值 | 敌人范围 | 说明 |
|------|-----------|----------|------|
| 暴击率 | 5% | 0 ~ 30% | 实际暴击率受装备加成 |
| 连击率 | 1% | 0 ~ 30% | 实际连击率受装备加成 |
| 最大连击次数 | 1次 | 1 ~ 3次 | 实际值受装备加成 |
| 障碍抗性 | 0% | 0 ~ 30% | 抵抗障碍技能的概率 |
| 忽视障碍异常 | 0% | 0 ~ 30% | 增加障碍技能成功率 |

#### 暴击计算
- **实际暴击率** = 玩家暴击率 + 所有装备强力词条暴击率（受最大暴击率上限限制）
- **暴击伤害** = 基础伤害 × 暴击倍数(1.5)

#### 连击计算
- **实际连击率** = 玩家连击率 + 所有装备强力词条连击率（受最大连击率上限限制）
- **连击概率衰减**：每次连击后概率衰减至90%
- **连击伤害递减**：每次连击伤害为前次的80%

## 战斗系统

### 回合流程

1. **先手判定**：根据速度属性点判断，速度高者先手
2. **行动执行**：行动方执行操作（普攻/技能/道具/防御/逃跑）
3. **结算**：计算伤害、状态效果
4. **切换**：行动方切换
5. **循环**：重复直到任意一方血量≤0

### 玩家行动

| 行动类型 | 效果 | 消耗 |
|----------|------|------|
| 物理攻击 | 对目标造成物理伤害 | 无 |
| 法术攻击 | 对目标造成法术伤害 | 法力值 |
| 防御 | 本回合减伤50% | 无 |
| 道具 | 使用背包道具恢复生命/法力 | 道具数量 |
| 逃跑 | 退出战斗 | 无 |

### 敌方行动

- **技能使用**：50%概率使用技能（如果有技能可用）
- **道具使用**：血量≤30%时概率使用回血道具
- **技能伤害** = 技能基础伤害 + 敌人魔法攻击 × 0.5

### 障碍技能系统

| 技能 | 成功率 | 持续回合 | 效果 |
|------|--------|----------|------|
| 封印术 | 小概率(30%) | 5回合 | 目标每回合跳过行动 |
| 中毒术 | 中概率(50%) | 5回合 | 每回合扣减8%生命+5%法力 |
| 冰冻术 | 大概率(70%) | 5回合 | 免疫伤害，但无法行动 |
| 混乱术 | 小概率(30%) | 5回合 | 随机攻击不分敌我 |

#### 混乱状态规则
- 随机选择目标（玩家、宠物或友方）
- 只能使用物理或法术攻击（50%概率）
- 法术攻击消耗法力，但法力不足时仍可使用

#### 冰冻状态规则
- 无法受到任何伤害
- 无法被施加除冰冻外的任何buff/debuff
- 冰冻术会覆盖其他所有buff
- 无法被治疗或恢复法力

#### 障碍成功率计算
**实际成功率** = 基础成功率 × (1 - 目标障碍抗性/100 + 施法者忽视障碍异常/100)

### Buff/Debuff显示
- 战斗面板显示敌人当前的buff/debuff
- 显示效果名称和剩余回合数
- 同类buff只会显示最新的一个（覆盖机制）

### 目标选择系统
- 点击攻击敌人时可选择具体目标
- 点击技能按钮时可选择目标
- 普通攻击和技能攻击都支持目标选择
- 有默认目标，但可自定义选择

## 地图与敌人系统

### 地图等级系统

- 地图等级可在地图界面设置
- **敌人属性** = 基础属性 × (1 + 1.3 × (地图等级 - 1))
- **技能使用限制**：
  - 地图等级 < 6：敌人不能使用辅助技能
  - 地图等级 < 9：敌人不能使用障碍技能

### 敌人生成

- 所有类型敌人都有概率出现在地图
- 敌人数量：3~10个
- 敌人类型区分物理攻击和法术攻击
- 敌人在地图上有独立的显示位置

| 4 | 史诗BOSS | 史诗 | 史诗挑战，奖励丰厚 |
| 5 | 传说BOSS | 传说/神话 | 终极挑战，最强大BOSS |

- **刷新数量**：每张地图刷新2个BOSS
- **单独配置**：BOSS单独配置，不参与全地图敌人数量计算
- **属性倍率**：BOSS等级属性提升系数1.8倍（比普通敌人更高）
- **位置显示**：BOSS在地图上有特殊位置，不与普通敌人重叠

- **特殊外观**：BOSS角色更大，有独特的颜色和光晕效果

- **血量条**：BOSS有特殊的血量条样式

- **金币倍数**：BOSS金币奖励×3倍
- **经验倍数**：BOSS经验奖励×2.5倍

## 地图移动系统

### 点击移动功能
- **地图点击**：点击地图非敌人区域，玩家会向该位置移动
- **目标位置**：计算点击位置在地图区域的百分比坐标
- **边界限制**：限制移动范围在地图边界内（0-100%）

### 平滑移动实现
- **动画帧**：使用requestAnimationFrame实现平滑移动
- **移动速度**：每帧移动0.5%的地图距离
- **距离计算**：计算当前位置与目标位置的距离
- **单位向量**：根据距离计算移动的单位向量
- **逐步移动**：每帧向目标位置移动一步
- **到达判定**：距离小于移动速度时直接到达目标位置

### 移动中重新选择目标
- **立即停止**：在移动过程中点击新位置会立即停止当前移动
- **重新计算**：清除原动画帧，从当前位置开始计算新路径
- **更新目标**：设置新的目标位置并开始新的移动动画

### 目标指示器
- **位置显示**：在点击的目标位置显示一个指示器
- **脉冲动画**：指示器有呼吸般的脉冲动画效果
- **样式设计**：半透明圆形，蓝色边框，背景淡蓝
- **自动隐藏**：到达目标位置后指示器自动隐藏

### 键盘移动
- **方向键支持**：支持上/下/左/右方向键和WASD键移动
- **停止平滑**：按下键盘移动时会停止当前的平滑移动
- **传统移动**：键盘移动使用传统的步进式移动
- **移动距离**：每次键盘移动10%的地图距离

### 战斗触发
- **敌人接触**：玩家移动过程中靠近敌人会触发战斗
- **安全距离**：距离小于60像素时触发战斗
- **地图敌人**：普通敌人和BOSS都可能触发战斗

## 战斗失败惩罚系统

### 惩罚计算方式

战斗失败时会扣除一定的金币和经验值，惩罚值基于以下因素：

- **等级差**：玩家等级与地图等级的差值的绝对值
- **敌人数量**：战斗中敌人的总数量

### 惩罚公式

```
基础惩罚 + 等级差 × 等级差系数 + 敌人数量 × 敌人数量系数
```

### 惩罚配置

| 配置项 | 数值 | 说明 |
|--------|------|------|
| 金币基础惩罚 | 5 | 战斗失败的基础金币惩罚 |
| 经验值基础惩罚 | 10 | 战斗失败的基础经验值惩罚 |
| 等级差系数 | 3 | 每差1级惩罚增加的数值 |
| 敌人数量系数 | 2 | 每多1个敌人惩罚增加的数值 |
| 地图等级高于玩家时惩罚降低比例 | 0.5 | 地图等级高于玩家等级时，惩罚降低50% |

### 惩罚上限

为了防止惩罚过大，设置了以下上限：

- **金币惩罚上限**：不超过当前金币的 20%
- **经验值惩罚上限**：不超过升级到下一级所需经验的 10%

### 惩罚调整机制

根据等级关系的不同，惩罚会有所调整：

- **玩家等级 ≥ 地图等级**：使用正常惩罚值
- **地图等级 > 玩家等级**：惩罚值降低 50%，鼓励玩家挑战高等级地图

### 特殊规则

- 当金币或经验值为 0 时，不再扣除相应的惩罚
- 惩罚不会将金币或经验值扣除到负数
- 战斗日志中会显示损失的金币和经验值数量

## 技能强化系统

### 强化机制
- **最大等级**：每个技能最大可强化300级
- **效果**：降低该技能最终法力消耗Math.floor(技能等级/10)%
- **强化费用**：基础费用50 + 当前强化等级×10
- **永久强化**：技能强化后永久生效

### 显示功能
- **技能面板**：在技能面板显示当前技能强化等级
- **调试面板**：在调试面板增加玩家/宠物的当前技能等级调试
- **消耗显示**：技能使用时显示实际消耗的法力值

## 商店系统

### 购买功能

| 商品类型 | 说明 |
|----------|------|
| 道具 | 生命恢复、法力恢复、百分比恢复 |
| 装备 | 无词条基础装备，等级和稀有度可配置 |

### 售卖功能

- **道具售价** = 购买价格 × 50%
- **装备售价** = 基础价格 × 等级 × 稀有度倍率 × 50%

### 重铸功能

| 重铸类型 | 费用计算 | 效果 |
|----------|----------|------|
| 全刷新 | 50 + 装备等级 × 1 | 刷新所有词条 |
| 单个基础词条 | 30 + 装备等级 × 0.5 | 刷新单个基础词条 |
| 单个强力词条 | 60 + 装备等级 × 0.5 | 刷新单个强力词条 |

### 宠物服务

- **属性重铸**：消耗金币随机改变宠物属性系数
- **技能学习**：消耗金币学习新宠物技能
- **技能强化**：消耗金币强化技能等级

- 存储内容：玩家数据、宠物数据、装备背包、道具背包、地图等级、地图敌人位置、地图BOSS位置

## 调试功能

### 调试面板

- **玩家属性修改**：可修改玩家属性点、暴击率、连击率、最大连击、技能等级
- **宠物属性修改**：可修改宠物属性系数、技能等级
- **装备生成**：可生成指定稀有度和等级的装备
- **战斗调试**：可在战斗中给单位施加buff
- **战斗重置**：可重置当前战斗
- **重置游戏**：清除存档并重新开始

## 胜负判定

- **玩家胜利**：所有敌人血量 ≤ 0
- **战斗失败**：玩家和宠物血量 ≤ 0
- **逃跑成功**：玩家主动退出战斗，返回地图
