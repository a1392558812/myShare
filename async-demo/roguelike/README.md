# Roguelike 肉鸽游戏 — 需求与设计文档

## 一、游戏架构说明

本游戏采用 **Canvas 全屏渲染 + HTML HUD 叠加** 架构：

- **Canvas 层**：承载所有游戏实体（玩家、敌人、弹幕、特效）的绘制，尺寸等于视口大小
- **HTML 层**：叠加在 Canvas 上方，负责 HUD（血条、经验条、技能栏）、升级选择面板、死亡界面、Boss 预警条的展示与交互
- **游戏主循环**：`requestAnimationFrame` 驱动，每帧执行 `update → render` 流程
- **数据驱动**：所有常量与数据结构集中于 `constants.js`，游戏逻辑仅消费常量，不硬编码数值
- **组件化**：使用 Vue 3 Composition API，逻辑按功能拆分为独立 composables
- **时间管理**：完全使用帧累积（`dt`）驱动，不使用 `setTimeout`/`setInterval`

## 二、坐标系统设计

### 逻辑坐标 vs 屏幕坐标

| 坐标系   | 说明                                         | 示例                        |
|----------|----------------------------------------------|-----------------------------|
| 逻辑坐标 | 世界空间中的绝对位置，地图逻辑上无限延伸       | 玩家初始 (0, 0)             |
| 屏幕坐标 | Canvas 画布上的像素位置，用于绘制与鼠标输入    | Canvas 中心 = (w/2, h/2)    |

### 转换公式

- **逻辑 → 屏幕**：`screenX = logicalX - cameraX + canvasWidth/2`
- **屏幕 → 逻辑**：`logicalX = screenX + cameraX - canvasWidth/2`
- Y 轴同理
- `toScreen` / `toLogical` 函数兼容 `CanvasRenderingContext2D` 和 `HTMLCanvasElement` 两种参数

### 镜头跟随

镜头始终以玩家为中心平滑跟随：

```
cameraX += (playerX - cameraX) * CAMERA_EASING
cameraY += (playerY - cameraY) * CAMERA_EASING
```

- `CAMERA_EASING = 0.08`：缓动系数，值越小跟随越平滑

### 软边界系统

为防止玩家无限跑图导致怪物无法追上，引入**软边界 + 渐进式惩罚**机制：

| 配置项          | 类型   | 默认值 | 说明                              |
|-----------------|--------|--------|-----------------------------------|
| `radiusX`       | number | 1.5    | X 轴边界半径倍率（× canvas.width） |
| `radiusY`       | number | 1.5    | Y 轴边界半径倍率（× canvas.height）|
| `tickInterval`  | number | 500    | 伤害判定间隔（ms，帧累积）         |
| `warningRatio`  | number | 0.2    | 预警告触发比例（距边界 ≤ 20%）     |

**三档渐进惩罚**（zones，threshold 为越界深度倍率 × canvas.width）：

| 档位 | threshold | 伤害（/500ms） | 减速   | 说明     |
|------|----------|---------------|--------|----------|
| 1    | 0        | 1% 最大生命   | 10%    | 刚越界   |
| 2    | 0.3      | 3% 最大生命   | 25%    | 深入     |
| 3    | 0.6      | 6% 最大生命   | 50%    | 致命深度 |

**关键设计**：

- **倍率自适应**：边界距离 = 倍率 × canvas 实际尺寸，不同窗口大小下体感一致
- **帧驱动**：完全使用 `dt` 累加计时器，不使用 `setTimeout`/`setInterval`
- **无敌保护**：无敌技能 / 调试无敌状态下跳过扣血但仍保留减速
- **预警告**：接近边界（≤20% 半径）时开始显示红色渐隐提示
- **Slow 叠加**：边界减速与 deathZone / Boss 减速乘法叠加，互不冲突
- **视觉特效**：
  - 边界线：三层叠加（外发光 18px + 警示条纹 7px + 内亮线 3px），红色脉冲
  - 警告特效：四边向内线性渐变 shadow（屏幕边缘 → 内部），dangerLevel 递增
  - 警告文字：三级递增（"危险边界" → "严重警告" → "致命边界"），红色发光

## 三、模块划分

| 模块               | 职责                                       |
|--------------------|--------------------------------------------|
| 输入处理           | 键盘（WASD移动 / 技能按键）与鼠标（方向 + 按住持续射击） |

| 镜头系统           | 缓动跟随玩家，坐标转换                        |

| Boss 系统          | Boss 触发检测、阶段转换、多阶段攻击模式、掉落生成     |
| 弹幕系统           | 箭矢 / 追踪弹幕 / 敌人弹幕的飞行与碰撞、穿透攻击逻辑         |
| 特效系统           | 近战劈斩弧、冰冻圈、吸血光环、无敌光环、冲刺残影的渲染与衰减       |
| 事件系统           | 随机事件生成、祭坛 / 神龛 / 宝箱 / 石碑的交互 |

| 升级系统           | 经验积累、等级判断、技能选择面板（无上限升级）  |
| HUD 渲染           | HTML 叠加层：血条、经验条、技能栏、时间、击杀数、Boss 预警条   |
| 渲染管线           | 背景网格 → 事件 → 特效 → 吸血光环 → 敌人 → 弹幕 → 玩家 → HUD |

### 5.2 成长计算公式

每个技能的每个属性有独立成长配置，支持三种成长模式：

| 模式 | 配置字段 | 公式 | 说明 |
|------|---------|------|------|
| 线性递增 | `increment` + `cap` | `value = min(base + increment × (level-1), cap)` | cap=null 不封顶 |
| 指数增长 | `ratio` + `cap` | `value = min(base × ratio^(level-1), cap)` | cap=null 不封顶 |
| 指数递减 | `ratio` (<1) + `cap` | `value = max(base × ratio^(level-1), cap)` | 冷却/持续等 |

## 六、敌人系统

### 6.1 敌人 HP 缩放公式

普通敌人使用线性缩放公式：

```
HP(playerLevel) = baseHP × (1 + ENEMY_HP_SCALE_RATE × (playerLevel - 1))
```

| 参数 | 值 | 说明 |
|------|-----|------|
| `ENEMY_HP_SCALE_RATE` | 0.06 (6%) | 每级 HP 增长比例 |
| `ENEMY_ATTACK_SCALE_RATE` | 0.05 (5%) | 每级攻击力增长比例 |

精英敌人使用固定增长公式：

```
HP(playerLevel) = baseHP + hpGrowth × max(0, playerLevel - 7)
```

## 七、Boss 系统

### 7.1 Boss 触发条件

- **首次触发**：游戏时间 ≥ 120 秒，且玩家等级 ≥ 3
- **后续触发**：上次 Boss 被击败后，冷却 180 秒，且玩家等级 ≥ 上次 Boss 等级 + 2
- **提前预警**：
  - 首次 Boss：提前 30 秒显示 HUD 预警条
  - 后续 Boss：满足条件后立即显示 HUD 预警条

### 7.2 Boss 受击保护

为避免 Boss 被瞬间秒杀，Boss 拥有以下保护机制：

| 保护机制 | 参数 | 值 | 说明 |
|---------|------|-----|------|
| 受击无敌帧 | `BOSS_HIT_INVINCIBLE_TIME` | 150ms | 受击后短时间内免疫其他伤害 |
| 单次伤害上限 | `BOSS_MAX_DAMAGE_RATIO` | 8% | 单次伤害不超过最大 HP 的 8% |

**无敌帧设计意图**：阻止同一帧内多箭/多火球同时命中将所有伤害叠加。150ms 无敌帧 ≈ 玩家最快攻击间隔（箭 100ms cd），保证每箭间隔至少 150ms 才生效一次。

**视觉反馈**：
- 正常受击（伤害生效）：红闪 → 白闪（`hitFlash`）
- 无敌期间受击（伤害无效）：青蓝色闪（`_hitInvincibleFlash`），提示玩家"命中了但伤害被抵抗"

## 十、升级系统

### 10.1 经验获取

- **击杀敌人**：获得敌人 `expReward` 经验值
- **精英敌人**：普通精英 ×1.5、稀有精英 ×2.5
- **Boss**：获得 Boss `xpReward` 经验值

### 10.2 经验等级公式

- **Lv 1~15**：固定表（见 `EXP_LEVEL_TABLE`）
- **Lv 16+**：线性增长
  - Lv15→16 增量 = 3000
  - 每升一级，下一级增量固定 +200
  - 总阈值为 O(n²) 多项式增长

### 10.3 升级流程

1. 经验达到升级阈值 → 等级 +1
2. 立即调用 `recalcPassiveBuffs()`（强身健体效果即时生效）
3. 弹出升级选择面板（3 个选项，从技能池随机抽取）
4. 玩家选择后，技能升级或新技能解锁

## 十一、伤害计算链路

### 11.1 完整公式

```
最终伤害 = 技能基础伤害(等级) × (1 + 基础攻击力加成) × (1 + Buff加成) × (1 + 无敌伤害加成)
```

| 环节 | 来源 | 说明 |
|------|------|------|
| 技能基础伤害 | `calcSkillValue(skill.damage, level)` | 应用成长公式 |
| 基础攻击力加成 | 强身健体 `attackBonus` | 全局乘算 |
| Buff 加成 | 力量祭坛 `attackBoost` (+50%) | 临时乘算 |
| 无敌伤害加成 | `damageBoost` `increment: 0.02` | 线性增长 |

### 11.2 穿透攻击公式

```
实际穿透次数 = min(calcSkillValue(penetration, level), cap=5)
```

每穿透一个敌人，箭矢继续飞行寻找下一个目标。

### 11.3 分裂箭伤害

```
分裂箭伤害 = 主箭伤害 × 0.7
```

### 11.4 吸血量公式

```
吸血量 = 光环伤害 × lifestealPercent
```

`lifestealPercent` 最高 80%（Lv12 封顶）。

## 十二、游戏流程

### 12.1 开始游戏

1. 点击 "开始游戏" 按钮
2. 进入游戏主循环
3. 玩家出生在 (0, 0)，初始拥有 **射箭** 技能

### 12.2 游戏进行

1. **移动**：WASD 或方向键
2. **攻击**：鼠标按住持续射箭，其他技能自动触发或手动释放
3. **升级**：经验达到阈值后弹出升级面板，选择技能升级或新技能解锁
4. **事件**：Lv8 后随机生成事件，靠近交互
5. **Boss**：满足条件后触发 Boss 战

### 12.3 游戏结束

1. 玩家 HP ≤ 0 → 显示死亡界面
2. 显示本次游戏数据（击杀数、等级、游戏时长）
3. 点击 "重新开始" 返回开始界面
