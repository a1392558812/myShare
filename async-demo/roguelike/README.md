# Roguelike 肉鸽游戏 — 需求文档

## 一、游戏架构说明

本游戏采用 **Canvas 全屏渲染 + HTML HUD叠加** 架构：

- **Canvas 层**：承载所有游戏实体（玩家、敌人、弹幕、特效）的绘制，尺寸等于视口大小
- **HTML 层**：叠加在 Canvas 上方，负责 HUD（血条、经验条、技能栏）、升级选择面板、死亡界面的展示与交互
- **游戏主循环**：`requestAnimationFrame` 驱动，每帧执行 `update → render` 流程
- **数据驱动**：所有常量与数据结构集中于 `constants.js`，游戏逻辑仅消费常量，不硬编码数值

### 文件结构

```
async-demo/roguelike/
├── index.vue      — 游戏主组件（Canvas + HUD + 交互逻辑）
├── constants.js   — 全部常量与数据结构定义（含技能成长公式）
└── README.md      — 本需求文档
```

## 二、坐标系统设计

### 逻辑坐标 vs 屏幕坐标

| 坐标系   | 说明                                         | 示例                        |
|----------|----------------------------------------------|-----------------------------|
| 逻辑坐标 | 世界空间中的绝对位置，地图逻辑上无限延伸       | 玩家初始 (0, 0)             |
| 屏幕坐标 | Canvas 画布上的像素位置，用于绘制与鼠标输入    | Canvas 中心 = (w/2, h/2)    |

### 转换公式

- **逻辑 → 屏幕**：`screenX = logicalX - cameraX + canvasWidth/2`
- **屏幕 → 逻辑**：`logicalX = screenX + cameraX - canvasWidth/2`
- Y 轴同理
- `toScreen` / `toLogical` 函数兼容 `CanvasRenderingContext2D`（取 `.canvas.width`）和 `HTMLCanvasElement`（取 `.width`）两种参数

### 镜头跟随

镜头始终以玩家为中心平滑跟随：

```
cameraX += (playerX - cameraX) * CAMERA_EASING
cameraY += (playerY - cameraY) * CAMERA_EASING
```

- `CAMERA_EASING = 0.08`：缓动系数，值越小跟随越平滑

#### 成长配置 (`growth`)

每个技能拥有 `growth` 对象，定义各字段的成长比例与硬上限。**技能无等级上限（移除 `maxLevel`），可无限升级**。

`growth` 结构：每个字段 = `{ ratio: number, cap: number | null }`

**计算公式**：`skillValue = clamp(base × ratio^(currentLevel - 1), cap)`（由 `calcSkillValue()` 函数统一计算）

- `ratio > 1` → 递增型（伤害、范围、持续时间等），`cap` 为硬上限（不可超过）
- `ratio < 1` → 递减型（冷却时间，越升级越短），`cap` 为硬下限（不可低于）
- `cap = null` → 无限制

## 五、技能系统设计

### 5.1 初始技能

玩家初始仅拥有 **射箭**（Lv1 即解锁），朝鼠标方向发射箭矢。

### 5.2 技能解锁条件

| 等级 | 可解锁新技能     |
|------|-----------------|
| 1    | 近战劈斩         |
| 1    | 追踪弹幕         |
| 1    | 冰冻定身         |
| 1    | 吸血光环         |

### 5.3 升级路径（无上限成长）

每次升级时弹出选择面板，提供最多 3 个选项：
- **新技能解锁**：尚未拥有的且 unlockLevel ≤ 当前等级 的技能
- **已有技能升级**：技能提升 1 级，**无等级上限**

升级后各属性值由公式 `base × ratio^(level-1)` 计算，并受硬上限约束。具体计算方式：

```
calcSkillValue(base, { ratio, cap }, currentLevel)
  value = base × ratio ^ (currentLevel - 1)
  若 ratio ≥ 1 且 cap != null → value = min(cap, value)
  若 ratio < 1 且 cap != null → value = max(cap, value)
```

示例：
- 射箭伤害 Lv1=10，Lv2=10×1.35=13.5，Lv5≈33，Lv10≈76，无上限
- 射箭冷却 Lv1=300ms，Lv5≈197ms，Lv10→clamp 120ms（到达下限）
- 追踪弹幕范围 Lv1=400px，因 ratio=1.08 且 base 已超上限 → 直接 clamp 360px

### 5.4 技能使用方式

| 技能       | 触发方式                          | 效果机制                               |
|-----------|----------------------------------|----------------------------------------|
| 射箭       | 鼠标按住持续射击 / 空格键单次       | 箭矢朝鼠标逻辑坐标飞行，命中即销毁；冷却结束自动射下一箭 |
| 近战劈斩   | **自动触发**（范围内有敌人时）      | 以玩家为中心、朝鼠标方向的扇形范围伤害   |
| 追踪弹幕   | 数字键 3 / 技能栏点击              | 朝最近敌人发射追踪弹幕，持续修正飞行方向    |
| 冰冻定身   | 数字键 4 / 技能栏点击              | 范围内敌人定身 + 减速，附带少量伤害        |
| 吸血光环   | 数字键（对应位置）/ 技能栏点击      | 激活持续光环，每秒对范围内敌人造成伤害并回复自身 |

## 六、敌人 AI 行为状态机

### 6.1 状态定义

| 状态     | 条件                          | 行为                               |
|---------|-------------------------------|------------------------------------|
| 追踪    | 距玩家 > attackRange / 射程外且无技能可用 | 朝玩家直线移动                       |
| 技能攻击 | 远程型：距玩家 ≤ skillRange 且冷却完成   | 发射弹幕，进入冷却                   |
| 近战攻击 | 近战型/混合型：距玩家 ≤ attackRange 且冷却完成 | 单次固定伤害，进入冷却；冷却期间仍追踪玩家 |
| 定身    | 被冰冻技能命中                   | 不移动、不攻击，等待冰冻计时结束        |
| 死亡    | hp ≤ 0                         | 移除实体，给予玩家经验                 |

### 6.2 各类型行为差异

#### 近战型 (melee)
1. 一直朝玩家追踪移动，**攻击范围内也不停下**
2. 进入 attackRange 且冷却结束 → 立即打一下固定伤害，然后进入冷却
3. 冷却期间继续追踪，冷却结束再判断是否攻击
4. 离开攻击范围 → 冷却计时器归零，下次追入又立即打
5. 无远程技能

#### 远程型 (ranged)
1. 朝玩家追踪移动
2. 进入 skillRange 后发射弹幕，进入冷却
3. 冷却期间继续移动保持距离
4. 无近战能力

#### 混合型 (hybrid)
1. 远距离时追踪移动
2. 进入 skillRange 且冷却完成 → 发射弹幕
3. 进入 attackRange 且近战冷却完成 → 近战攻击
4. 技能优先级：远程 > 近战

### 6.3 状态转换图

```
[刷新] → 追踪
追踪 → 近战攻击 (近战型：距 ≤ attackRange, 冷却结束, 立即攻击后进入冷却)
追踪 → 技能攻击 (远程型/混合型：距 ≤ skillRange, 冷却完成)
近战攻击 → 追踪 (冷却期间继续追踪，冷却结束再判断)
技能攻击 → 追踪 (冷却中)
任意 → 定身 (被冰冻命中)
定身 → 追踪 (冰冻结束)
任意 → 死亡 (hp ≤ 0)
```

## 七、后续扩展规划

### 7.1 短期扩展（可快速实现）

- **道具系统**：击败敌人随机掉落血瓶、加速靴等道具
- **迷你地图**：右上角显示玩家位置与周围敌人分布
- **波次模式**：按波次递增难度，而非纯时间递增
- **Boss 敌人**：特殊大型敌人，多阶段攻击模式

### 7.2 中期扩展

- **地图地形**：引入障碍物（墙壁、树木），影响移动路径
- **装备系统**：击败 Boss 获得装备，改变玩家外观与属性
- **技能组合**：两个技能联动触发组合效果（如冰冻 + 射箭 = 冰箭）
- **被动技能**：升级时可选择被动增益（如移动速度 +10%、自动回血）

### 7.3 长期扩展

- **多人协作**：WebSocket 支持多人在线合作
- **房间制**：玩家进入不同地图房间，房间间有过渡动画
- **商店系统**：使用击杀金币购买永久属性提升
- **存档系统**：localStorage 存储玩家进度，支持断线续玩
- **音效与 BGM**：Web Audio API 加入攻击音效、环境音乐
