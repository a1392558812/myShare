# Roguelike 随机性与 Boss 机制：设计与改版思路

## 一、现状诊断

| 维度 | 现状 | 问题 |
|------|------|------|
| **敌人种类** | 仅 3 种（近战/远程/混合），行为模式固定 | 玩 5 分钟就体验完了所有内容 |
| **敌人个体** | 同类型完全一致，无变体 | 没有"精英怪"的惊喜/惊吓感 |
| **Boss 战** | 无 | 缺少高潮节点和节奏变化 |
| **环境** | 空旷无限平面 | 没有地形交互、随机事件 |
| **掉落** | 仅血瓶+金币 | 没有功能性道具/临时强化 |
| **节奏** | 平滑刷怪→索引难度 | 缺少"爆发点" |

---

## 二、改版总体架构：四层随机性模型

```
┌─────────────────────────────────────────────────────┐
│                   Layer 4: Boss 波次                  │
│  定时触发 → 全屏警告 → 多阶段 Boss 战 → 特殊掉落       │
├─────────────────────────────────────────────────────┤
│                   Layer 3: 随机事件                   │
│  场上概率出现 → 祭坛/宝箱/诅咒石碑 → 玩家抉择          │
├─────────────────────────────────────────────────────┤
│                   Layer 2: 精英词缀                   │
│  敌人诞生时 X% 概率携带词缀 → 外观+能力双异化          │
├─────────────────────────────────────────────────────┤
│                   Layer 1: 敌人多样性                  │
│  扩充类型表 + 同类型属性浮动 (±20%)                   │
└─────────────────────────────────────────────────────┘
```

## 三、Layer 1 — 敌人多样性增强（改动量：小）

### 3.1 同类型属性浮动

在 `useEnemySpawner.spawnEnemy()` 中，生成敌人属性时加 ±15%~20% 的随机浮动：

```
hp    = scaledHp   * random(0.85, 1.15)
speed = attrs.speed * random(0.9, 1.1)
size  = attrs.size  * random(0.9, 1.2)  // 大小微变，视觉差异
```

这样即使同一类型敌人，每次遭遇也略有不同，不会出现"两只远程怪行为完全一致"的无聊感。

### 3.2 扩充敌人类型（建议新增 3~4 种）

| 类型 | 特点 | 定位 |
|------|------|------|
| **自爆怪** `bomber` | 低血量高速度，接近玩家后自爆造成 AoE 伤害 | 压迫走位 |
| **召唤师** `summoner` | 远程，不直接攻击，定期召小怪 | 优先击杀目标 |
| **冲锋者** `charger` | 中血量，周期性直线冲锋（短时间超高速），撞到玩家造成大量伤害+击退 | 考验反应 |
| **护盾兵** `shielder` | 跟随最近的敌人，给它提供减伤光环 | 改变击杀顺序 |

每个新类型需要在 `constants.js` 的 `ENEMY_TYPE_TABLE` 注册，并在 `useEnemy.js` 的 `updateEnemies` 中添加对应 AI 逻辑。

# 精英怪系统设计文档

## 一、设计目标

- 打破 Lv7 后玩家"割草"节奏，重建挑战曲线
- 精英怪为**独立敌人单位**，非现有敌人词缀叠加
- 每种精英有独立属性配置 + 按玩家等级线性成长
- 视觉上有明显标识，玩家能一眼分辨类型和稀有度

---

## 二、出现时机与权重

### 等级分段

| 玩家等级 | 精英种类 | 总权重 | 精英占比 | 说明 |
|---------|---------|-------|---------|------|
| Lv 1~7 | 无 | 15 | 0% | 学习期，熟悉基础敌人 |
| Lv 8~12 | 疾风 + 血牛 | 15+4=19 | ~21% | 普通精英出场 |
| Lv 13+ | 全部 4 种 | 15+6=21 | ~29% | 稀有精英加入 |

### 权重分配（Lv13+ 最终版）

```js
// ENEMY_TYPE_TABLE 最终结构
[
  { type: 'melee',    weight: 4 },
  { type: 'ranged',   weight: 3 },
  { type: 'hybrid',   weight: 2 },
  { type: 'bomber',   weight: 2 },
  { type: 'summoner', weight: 1 },
  { type: 'charger',  weight: 2 },
  { type: 'shielder', weight: 1 },
  // ── 精英怪 ──
  { type: 'eliteWind',  weight: 2 },  // 疾风（普通）
  { type: 'eliteBlood', weight: 2 },  // 血牛（普通）
  { type: 'elitePriest', weight: 1 }, // 牧师（稀有）
  { type: 'eliteVenom',  weight: 1 }, // 毒虫（稀有）
]
```

## 三、属性成长公式

精英怪从 **Lv7 后**开始出现，属性随玩家等级线性增长：

```js
// 伪代码
const growth = Math.max(0, playerLevel - 7)
eliteStat = baseStat + growth * growthPerLevel
```

- `playerLevel <= 7`：不生成精英（Spawner 层面过滤）
- `playerLevel > 7`：按上述公式成长，无指数爆炸

## 四、4 种精英怪属性配置

### 4.1 疾风（普通精英 · 青色尾迹）

**定位**：高速，较脆，逼玩家预判走位，不能站桩输出

| 属性 | 基础值 | 每级成长 | Lv7 | Lv15 | Lv30 |
|------|--------|---------|------|------|------|
| maxHp | 60 | +8 | 60 | 124 | 244 |
| attack | 12 | +2 | 12 | 28 | 58 |
| speed | 4.2 | 0（固定） | 4.2 | 4.2 | 4.2 |
| expReward | 35 | — | ×1.5 | ×1.5 | ×1.5 |
| size | 40 | 0 | 40 | 40 | 40 |

**视觉标识**：
- 主体色：`#06b6d4`（青色）
- 尾迹：每帧在身后生成 3~4 个渐隐青色圆点（`effects.js` 粒子系统）
- 名称前缀：「**疾风** 近战武士」

**AI 行为**：同普通近战敌人，无特殊技能，仅靠高速逼近施压

---

### 4.2 血牛（普通精英 · 明显更大）

**定位**：肉，伤害高，不能硬扛，优先处理

| 属性 | 基础值 | 每级成长 | Lv7 | Lv15 | Lv30 |
|------|--------|---------|------|------|------|
| maxHp | 180 | +20 | 180 | 340 | 640 |
| attack | 20 | +3 | 20 | 44 | 89 |
| speed | 2.0 | 0（固定） | 2.0 | 2.0 | 2.0 |
| expReward | 45 | — | ×1.5 | ×1.5 | ×1.5 |
| size | 60（=40×1.5） | 0（固定） | 60 | 60 | 60 |

**视觉标识**：
- 主体色：`#991b1b`（暗红）
- 尺寸明显更大（1.5x），碰撞体积同步放大
- 名称前缀：「**血牛** 近战武士」

**AI 行为**：同普通近战敌人，无特殊技能

---

### 4.3 牧师（稀有精英 · 血色光环）

**定位**：辅助型，本身威胁低，但给范围内友军回血，必须优先击杀

| 属性 | 基础值 | 每级成长 | Lv7 | Lv15 | Lv30 |
|------|--------|---------|------|------|------|
| maxHp | 70 | +10 | 70 | 150 | 300 |
| attack | 8 | +1.5 | 8 | 20 | 42 |
| speed | 1.8 | 0（固定） | 1.8 | 1.8 | 1.8 |
| expReward | 60 | — | ×2.5 | ×2.5 | ×2.5 |
| 回血频率 | 2000ms | 0（固定） | — | — | — |
| 回血量 | 8 | +2 | 8 | 24 | 54 |
| 光环半径 | 130px | 0（固定） | — | — | — |
| size | 44（略大） | 0 | 44 | 44 | 44 |

**视觉标识**：
- 主体色：`#7f1d1d`（暗血红） + `#fca5a5`（血色光环）
- 光环：半透明血色圆（130px 半径），跟随敌人位置持续渲染
- 名称前缀：「**牧师** 亡灵牧师」

**AI 行为**：
- 同远程敌人，保持与玩家距离
- 每 2000ms 扫描 130px 范围内所有 `!dead` 敌人（含自己），回复 `healAmount` HP（不超过 maxHp）
- 回血事件写入 `battleLog`（"牧师为 XX 回复了 YY HP"）

---

### 4.4 毒虫（稀有精英 · 毒液缠绕）

**定位**：控制场地，逼走位，地面污染持续造成伤害

| 属性 | 基础值 | 每级成长 | Lv7 | Lv15 | Lv30 |
|------|--------|---------|------|------|------|
| maxHp | 50 | +6 | 50 | 98 | 188 |
| attack（毒弹伤害） | 10 | +2 | 10 | 26 | 56 |
| speed | 2.5 | 0（固定） | 2.5 | 2.5 | 2.5 |
| expReward | 55 | — | ×2.5 | ×2.5 | ×2.5 |
| 毒弹间隔 | 3000ms | 0（固定） | — | — | — |
| 预警圈时长 | 800ms | 0（固定） | — | — | — |
| 地面毒区持续时间 | 5000ms | 0（固定） | — | — | — |
| 毒区伤害 | 2/秒 | +0.5/级 | 2 | 6 | 13.5 |
| 毒区半径 | 50px | 0（固定） | — | — | — |
| 毒区数量上限 | 无上限 | — | — | — | — |
| size | 40 | 0 | 40 | 40 | 40 |

**视觉标识**：
- 主体色：`#166534`（深绿） + `#4ade80`（毒绿）
- 毒液缠绕粒子：敌人周围持续漂浮绿色粒子
- 地面毒区：半透明绿色圆形区域，边缘有气泡粒子
- 名称前缀：「**毒虫** 毒液虫」

**AI 行为**：
1. 每 3000ms 向玩家当前位置发射一颗毒弹（弹速 4，比普通弹幕慢）
2. 毒弹发射时有 **800ms 预警圈**（绿色圆圈在地表显示），玩家可走位躲避
3. 毒弹落地后生成 `groundZone`（类型 `venom`）
4. 毒区持续 5000ms，每秒 tick 一次伤害（低伤害，逼走位）
5. 玩家在毒区内每秒扣血一次，伤害不走 `invincible`（因为是地面持续伤害，非敌人直接攻击）
6. 同位置多个毒区**不叠加伤害**

## 七、注意事项

1. **血牛尺寸固定 1.5x**，不随等级放大，避免后期屏幕被占满
2. **精英怪速度均不成长**，避免后期快到玩家无法反应
3. **牧师回血频率固定 2000ms**，光环半径固定 130px
4. **毒虫预警圈 800ms** 是硬数值，不给成长，保证玩家始终能反应
5. **毒区无数量上限**，但每个毒虫独立管理，新毒区覆盖旧毒区（每个毒虫最多保留最近 3 个毒区，防止数组无限增长）
6. **精英怪计入 `MAX_ENEMIES` 上限**（它们是独立敌人单位，占刷新名额）

## 五、Layer 3 — 随机事件与环境交互（改动量：中）

### 5.1 事件类型

| 事件 | 触发方式 | 效果 | 持续时间 |
|------|----------|------|----------|
| 🗿 **力量祭坛** | 玩家靠近按 F | 攻击力 +50% | 30s |
| 🔵 **速度神龛** | 玩家靠近按 F | 移速 +40%，闪避 20% | 20s |
| 📦 **宝箱** | 自动拾取 | 随机掉落 2-3 件道具 | 永久 |
| ⚠️ **诅咒石碑** | 玩家选择是否激活 | 敌人变强 30% 但掉落率×3 | 60s（高风险高回报） |
| 💀 **死亡区域** | 进入即生效 | 持续扣血 / 减速 | 区域内永久 |

### 5.2 生成规则

- 每 30~60 秒概率在地图随机位置生成 1 个事件
- 场上最多同时存在 3 个事件
- 祭坛/神龛有使用冷却（同类型 60s 内不重复生成）
- 事件出现在屏幕内任意位置（非边界处）

### 5.3 数据结构

```js
// 新状态文件
const worldEvents = ref([])
// 每个事件:
{
  id: 'shrine_power',
  type: 'shrine',        // shrine | chest | curse | zone
  name: '力量祭坛',
  icon: '🗿',
  x, y,
  radius: 40,
  effect: { ... },
  duration: 30000,
  elapsed: 0,
  used: false,           // 是否已激活（避免重复）
}
```

### 5.4 新增文件

建议新增 `composables/useWorldEvents.js`，负责事件的生成、生命周期、效果应用。

## 六、Layer 4 — Boss 波次系统（改动量：大）⭐⭐ 高光点

### 6.1 触发条件

不按"杀敌数"触发（那样会变成纯数值游戏），而是按 **游戏时间 + 玩家等级混合** 触发：

| Boss | 触发条件 | 冷却 |
|------|----------|------|
| **第 1 只 Boss** | 游戏时间 ≥ 2 分钟 + 玩家 Lv ≥ 3 | — |
| **后续 Boss** | 距上次 Boss 击败 ≥ 3 分钟 + 玩家 Lv 高于上次 +2 | 3min |

这样确保玩家有成长空间，同时保持节奏。

### 6.2 Boss 设计（建议 3 只，逐步解锁）

#### Boss 1: **暗影聚合体** `shadow amalgam`

```
HP: 300 + 30×(玩家等级-3)
体型: 3x 普通敌人
移动: 缓慢追踪 (speed 1.5)
```

**阶段 1（HP > 60%）**：
- 近战拍击（扇形 AoE，150px 范围）
- 发射 3 枚慢速追踪暗影球（伤害 15）

**阶段 2（HP ≤ 60%）**：
- 新增"暗影分身"：分裂为 3 个幻影，随机位置闪现，只有真身可受伤（真身有微弱金色轮廓）
- 暗影球数量增加到 5 枚

**阶段 3（HP ≤ 25%）**：
- 狂暴：移速翻倍，攻击冷却减半
- 地板出现周期性暗影波纹（需走位躲避）

#### Boss 2: **火焰核心** `inferno core`（解锁条件：玩家 Lv ≥ 8）

```
HP: 600 + 50×(玩家等级-8)
体型: 2.5x
移动: 中速 (speed 2.2)，但倾向于保持距离
```

**核心机制**：不直接追击，而是在场地中周期性召唤火柱（地面预警圈→0.8s 后喷发）。同时释放旋转火焰弹幕（8 方向）。

#### Boss 3: **虚空织法者** `void weaver`（解锁条件：玩家 Lv ≥ 12）

```
HP: 1000 + 80×(玩家等级-12)
体型: 2x
移动: 传送 (每 3s 瞬移到场地随机位置)
```

**核心机制**：全场布局——布置"虚空线"，两条虚空线之间的连线会造成持续伤害。Boss 同时释放减速场和追踪弹幕。玩家需要在有限的安全区域内战斗。

### 6.3 Boss 战流程

```
1. 触发条件满足
2. 全屏警告文字 "⚠️ 强大的敌人正在接近..."
3. 屏幕边缘变红脉冲 2 秒
4. 暂停普通刷新 5 秒
5. Boss 从屏幕中央诞生（有登场动画）
6. Boss 血条显示在屏幕顶部（独立 UI）
7. 战斗中普通刷新频率降低 50%
8. Boss 击败 → 大爆炸特效 + 掉落专属道具
9. 恢复普通刷新
```

## 八、关键技术要点

### 8.1 词缀系统与现有逻辑的集成

词缀主要在三个地方生效：
1. **属性修改**（spawnEnemy 时）：直接乘到 hp/speed/attack 上
2. **行为注入**（updateEnemies 时）：检查 `affixes` 数组决定额外行为
3. **死亡触发**（damageEnemy 时）：检查 `affixes` + 敌人 `dead === true` 时触发 on-kill 效果

不需要修改现有技能逻辑，完全解耦。

### 8.2 Boss 与现有刷新系统的关系

Boss 活跃期间：
- `handleSpawning` 中增加 `isBossActive` 检查，降低刷新频率
- 但不清空已有敌人，Boss 战期间小怪仍在场（增加压迫感）

### 8.3 随机性的"可控"

所有随机池都带权重，可以：
- 前 2 分钟锁定某些危险词缀（如不朽）
- 随游戏时间逐步解锁更多词缀
- Boss 词缀单独配置（如 Boss 不会带分裂，因为它本身就是单体挑战）

---

## 九、总结

这个方案的核心思路是 **"渐进式随机"**：不是一开始就扔所有东西给玩家，而是随游戏进程逐步扩大随机池。四个 Layer 各司其职：

| Layer | 体验价值 | 实现成本 |
|-------|----------|----------|
| 属性浮动 + 新敌人 | 缓解重复感 | 低 |
| **精英词缀** ⭐ | 每场遭遇都不同，核心 Rogue 体验 | 中 |
| 随机事件 | 探索乐趣，风险-回报抉择 | 中 |
| Boss 波次 | 节奏高潮，成就感 | 高 |

**建议先从 Phase 1+2 开始**——属性浮动和 5 个核心词缀，这就能让游戏发生质变。有了词缀系统的基础后，Boss 的部分 AI（阶段切换、特殊弹幕）也可以复用词缀的思路。
