# 常用git操作

## 指定文件提交到分支

1. git status 查看所有修改/新增的文件，确认要提交的文件路径：

```shell
On branch main
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   src/pages/login.vue  # 要提交的文件
        modified:   src/utils/request.js # 不提交的文件
  Untracked files:
  (use "git add <file>..." to include in what will be committed)
        src/components/avatar.vue # 新增的要提交的文件
```

2. 将指定文件加入暂存区

```shell
# 1. 提交单个文件（绝对/相对路径均可）
git add src/pages/login.vue
# 2. 提交多个指定文件，空格分开
git add src/pages/login.vue src/components/avatar.vue
# 3. 提交指定目录下的所有文件
git add src/pages/ # 提交 pages 目录下的所有修改文件
# 4. 排除文件：先加所有，再移除不要的
git add .
git reset HEAD src/utils/request.js # 移除不需要提交的文件
```

3. 提交暂存区的指定文件

```shell
 # 提交 main 分支
git push origin main
# 1.追加到最后一次提交,不修改说明,直接追加
git commit --amend
# 2.追加到最后一次提交,修改说明并追加
git commit --amend -m "feat(login): 优化登录页 + 补充表单校验"
# 强制推
git push origin main --force
```

## 切换到远程分支

1. 拉取最新的远程分支信息（必做）

```shell
# 拉取远程仓库的所有分支信息（不会修改本地代码，仅更新本地缓存）
git fetch origin
# 查看本地+远程所有分支（远程分支以 remotes/origin/ 开头）
git branch -a
```

2. 创建并切换到本地跟踪分支

```shell
# 简化写法（等价于 git checkout -b develop origin/develop）
git checkout origin/develop
# 或
git switch origin/develop
```

## 将开发内容暂存，开发新的任务，登新任务完成后再继续开发原有任务

1. 暂存当前未完成的开发内容

```shell
# 查看当前修改
git status
# 给要暂存的加识别标识备注
git stash save "登录功能：未完成验证码校验逻辑"
```

2. 工作区所有未提交修改被暂存，终端输出

```shell
# git stash 会暂存「已跟踪文件的修改」+「暂存区的修改（git add 过的）」；
Saved working directory and index state On main: 登录功能：未完成验证码校验逻辑
```

假如暂存需要新增的未跟踪文件（新建的 new-file.vue）

> -u 也是 --include-untracked的简写

```shell
git stash save -u "新增头像组件：未完成样式"
```

3. 切换分支开发新内容

```shell
git checkout feature/pay
# 开发的新东西
git add .
git commit -m "feat(pay): 完成支付按钮点击逻辑"
git push origin feature/pay
```

4. 恢复之前暂存的内容继续开发

```shell
# 1. 切回原分支（如 main 分支）
git checkout main
# 2. 查看所有暂存记录（确认要恢复的暂存项）
git stash list

# 输出示例：
# stash@{0}: On main: 登录功能：未完成验证码校验逻辑
# stash@{1}: On develop: 新增头像组件：未完成样式

# 恢复指定暂存
#   方式1：恢复暂存并保留暂存记录（可多次恢复）
git stash apply stash@{0} # 恢复第0个暂存项
#   方式2：恢复暂存并删除该暂存记录（恢复后无需保留）
git stash pop stash@{0}

# 删除指定暂存
git stash drop stash@{0}
# 删除所有暂存
git stash clear
# 推
```

5. 总结一盏式粘贴

```shell
# ========== 暂存未完成的登录功能
git status # 显示修改了 login.vue，新增了 verify.js
git stash save -u "登录功能：未完成验证码校验，新增 verify.js"
git status # 工作区干净

# ========== 切换分支开发支付功能
git checkout feature/pay
# 编写支付功能代码
git add src/pay/*.vue
git commit -m "feat(pay): 完成支付接口对接"
git push origin feature/pay

# ========== 切回原分支恢复暂存
git checkout main
git stash list # 查看暂存：stash@{0}: 登录功能...
git stash pop stash@{0} # 恢复并删除暂存
# 继续开发登录功能
git add login.vue verify.js
git commit -m "feat(login): 完成验证码校验逻辑"
git push origin main

# ========== 清理无用暂存，保险起见，防止自己楼了开发的东西
git stash clear # 若有其他无用暂存，清空，否则时间长乐自己也不知道这个功能补了没
```

## 去除某个文件的git追踪

1. 仅取消追踪，不删除本地文件

```shell
git rm --cached 文件名
```

2. 取消追踪整个文件夹

```shell
git rm --cached -r 文件夹名
```

## 撤销上一次的git add 与 git commit

```shell
git reset HEAD~1 --mixed
```

## 其他与总结

`git fetch origin` + `git branch -r` 查看远程所有分支
git branch -a 同时查看本地 + 远程分支，
git branch 查看本地所有分支
git branch -D develop 删除本地库develop
`git push origin --delete develop` + `git remote prune origin` 从远程库中删除develop分支

> -d 「安全删除」: 仅删除已合并到当前分支的本地分支 确认分支代码已合并，无未提交 / 未合并修改
> -D 「强制删除」: 忽略合并状态，直接删除分支 分支代码无需保留（如废弃的功能分支）、分支有未合并但无用的修改

git commit 提交
git commit -am "init" 提交并且加注释
git commit -a 提交当前repos的所有的改变
git commit -m "This is the message describing the commit" 添加commit信息

> git commit -am: 无需提前 git add，仅已跟踪文件的修改（新增文件不提交）
> git commit -m: 必须提前 git add 指定文件,仅暂存区的文件（可精准控制）

git commit -v 当你用－v参数的时候可以看commit的差异

git remote add origin git@192.168.1.119:ndshow
git remote show origin 显示远程库origin里的资源
git remote show 查看远程库

git push origin master 将文件给推到服务器上
git push origin master:develop
git push origin master:hb-dev 将本地库与服务器上的库进行关联

git checkout --track origin/dev 切换到远程dev分支
git checkout -b dev 建立一个新的本地分支dev
git checkout dev 切换到本地dev分支
git checkout -b localBranch origin/remoteBranch 创建并切换到新的本地分支，并设置跟踪远程分支：

git status 查看当前状态
git stash push 将文件给push到一个临时空间中
git stash pop 将文件从临时空间pop下来

git rm a.a 移除文件(从暂存区和工作区中删除)
git rm --cached a.a 移除文件(只从暂存区中删除)
git rm -f a.a 强行移除修改后文件(从暂存区和工作区中删除)
git rm 文件名(包括路径) 从git中删除指定文件
git rm [file name] 删除一个文件

git add .
git add [file name] 添加一个文件到git index
git merge origin/dev 将分支dev与当前分支进行合并
git clone git://github.com/schacon/grit.git 从服务器上将代码给拉下来
git config --list 看所有用户
git ls-files 看已经被提交的
git log 看你commit的日志
git diff 查看尚未暂存的更新
git diff --cached 或 $ git diff --staged 查看尚未提交的更新
