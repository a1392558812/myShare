# ai 一些名词

## 一代目：LLM（大语言模型）

2019-2020 左右开始兴起
最核心的功能，通过模型学习训练，解析语义、上下文关系和逻辑，返回内容
但是会瞎说，也非常依赖训练，训练成啥就说啥
// 知识截止于训练时间，无法获取实时信息；无法直接与外部工具交互，执行能力有限

## 二代目 1：RAG（检索增强生成）

解决 LLM 知识局限的关键技术，其核心是 "解耦知识存储与推理能力"，将信息检索与大模型生成能力结合的技术，通过外部知识库提升生成内容准确性，扩展模型知识边界 (使模型能突破时间限制，基于最新信息回答问题)
使得能够完整的完成 用户提问 → 检索相关知识 → 将知识提供给 LLM → LLM 基于可靠信息生成回答
很大程度的解决了早期类似于 chatgat 瞎说生成虚假信息的问题，确保回答准确性和时效性

## 二代目 2：MCP（模型上下文协议）

AI 模型与外部工具、数据源的交互方式，即原来的 ai 只会说，有了 mcp，相当于装上了手，解决了一个模型的多种能力，使 AI 能安全调用各类工具（API、数据库、计算资源），实现 "模型 + 工具" 的增强智能模式

## 三代目：AGENT（智能体）

是将 LLM 的认知能力与工具使用能力深度融合，以 LLM 为 "大脑"，MCP 为 "神经系统"，RAG 为 "知识库"，构建 "感知 - 规划 - 行动" 的自主闭环，从被动响应到主动规划，能自主拆解任务、调用工具、反思结果并调整策略，

## Agent 的演进历程：

1. 规则驱动 Agent 2019 年以前：基于预设规则执行固定任务
2. LLM 增强 Agent 2020 左右（以早期 openai，chatgpt 为代表的 ai 出图功能）：用大模型理解自然语言，指导行动
3. 自主 Agent 2023-2024 年左右开始到现在：（以现在豆包，AutoGPT，Claude 等为代表）具备完全自主决策能力，能解决开放域复杂问题，

- LLM 突破了语言理解的边界
- RAG 突破了知识范围的边界
- MCP 突破了工具交互的边界
- Agent 突破了自主行动的边界

| 技术  |    核心价值    |       解决的问题       |           在整体中的角色            |
| :---: | :------------: | :--------------------: | :---------------------------------: |
|  LLM  | 语言理解与生成 |   让机器理解人类语言   |    智能 "大脑"，提供核心认知能力    |
|  RAG  |  动态知识获取  |   解决知识滞后与幻觉   |    "外部知识库"，提供最新信息源     |
|  MCP  |   标准化交互   | 解决模型与工具集成难题 |  "万能接口"，连接 "大脑" 与 "四肢"  |
| Agent | 自主执行与规划 |  从被动响应到主动行动  | 完整智能体，实现 "思考 + 执行" 闭环 |
