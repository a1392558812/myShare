# AI-Skills

> AI Skills 就是咱们给 AI 写的 **「专业技能说明书 + 回答行为准则」**，用来强制 AI 锁定身份、明确会什么、擅长什么、该怎么回答、不做什么。它不是官方固定语法，而是大家约定俗成的`prompt 结构化`写法，通常用 `skill: / skills: XXXXXXXX` 开头，让 AI 快速识别：要它扮演哪个领域的大🐮，输出要符合什么标准。

## 什么是prompt

> Prompt 就是咱们发给 AI 的所有指令、问题、要求、角色设定噔噔的总称，简单说，就是咱们跟 AI 说的每一句话、给它定的所有规则，都叫 Prompt

## Skills与prompt

如果说prompt是中杯的话，那么Skills就是超大杯，是prompt的超级进化版

## AI Skills 有啥用

- 固定专业身份：不让 AI 乱跨界、不答非所问（比如只让它做资深前端，不聊历史、娱乐）
- 明确技术栈 / 能力：告诉它咱要的是「前端 + Uniapp + H5」，不是后端、运维、设计
- 规范输出格式：强制优先给代码、讲原理、给避坑、不废话
- 限定风格：资深高手、干练、实战向、不科普基础、直接给方案，不嘻嘻哈哈

## AI Skills 与 MCP纠缠

- AI Skills是咱给 AI 设定的专业能力、技术栈、擅长方向、回答规则（偏`硬实力 + 输出规范`）。
- MCP是Main Character Prompt（主角角色提示词），也有叫 Master Character Profile，是一整套完整的核心角色总设定。
- MCP 是完整角色档案（身份、经验、性格、语气、领域、禁忌、能力等等全部打包）
- AI Skills 是 MCP 里面的技能模块（专门写：会什么、擅长什么、怎么输出）

MCP 是完整角色，AI Skills 是 MCP 里描述专业能力加回答规则的核心部分。没有 Skills，MCP 只有身份，没有干活能力。没有 MCP，Skills 只是一段技能说明，缺少角色定位

## 一段标准 AI Skills 包含什么

- 身份定位：多少年经验、级别（资深 / 架构 / 专家）
- 核心技术栈：精通什么、主攻什么（Vue/React Uniapp H5 CSS JS/TS 跨端登）
- 擅长场景：解决什么问题（跨端、布局、动画、兼容、性能、线上 bug）
- 回答规则（最重要）：必须怎么答、禁止怎么答

## 举例

// 举例1

```shell
skill: 你是10年一线资深前端开发工程师，主攻移动端H5、Uniapp多端开发（H5/小程序/App），精通 HTML5/CSS3/JavaScript/TypeScript、Flex/Grid、渐变/动画、兼容适配、工程化与线上问题排查；熟悉 uni-app、移动端原生能力、跨端差异处理。
回答规则：
1. 只回答前端相关问题，不闲聊、不跑题；
2. 优先给出可直接复制运行的代码；
3. 附带核心原理 + 关键注意点/兼容坑；
4. 语言干练、实战向，不讲冗余基础概念；
5. 涉及Uniapp优先给跨端兼容方案。
```

// 举例2

```shell
skill: 资深前端，10年经验，主Uniapp/H5/JS/TS/CSS布局动画适配；只做前端解答，优先给可运行代码+原理+避坑，干练实战，不废话。
```

## 怎么用

- 直接把这段 Skills 放在提问的最前面，或者先发一次 Skills，再连续问问题，AI 就会全程保持这个角色与能力输出
- 一些ide现在已经支持 `.skill` 格式文件,直接配置或者下载别人的，比如：[Skills](https://github.com/vuejs-ai/skills)，[antfu/skills](https://github.com/antfu/skills)
