# 配置github-action构建打包

1. 新建： `.github/workflows/build.yml`

2. 修改下面配置

```yaml
name: 手动构建并提交到 build-dist 分支

# 仅手动触发（点一下才构建，一般不这样配置，这个是我自己单独的需要手动打包。）
on:
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: 拉取代码
        uses: actions/checkout@v4
        with:
          persist-credentials: false

      - name: 设置 Node.js 23.11.0
        uses: actions/setup-node@v4
        with:
          node-version: 23.11.0
          cache: "npm"

      - name: 安装依赖
        run: npm install --legacy-peer-deps

      - name: 打包项目
        run: npm run build

      # 创建部署目录并复制整个 dist 文件夹
      - name: 准备部署目录
        run: |
          mkdir -p deploy
          cp -r dist deploy/

      # 👇 核心：把打包产物提交到 build-dist 分支
      - name: 部署到 build-dist 分支
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          branch: build-dist # 目标分支
          folder: deploy # 使用包含 dist 文件夹的 deploy 目录作为部署目录
          clean: true # 清空旧文件
```

## action构建

1. 手动点击触发，不自动跑，（如果配置提交代码打包，那么打包会过于频繁）
2. 用 Node 23.11.0
3. 执行 npm run build
4. 自动把 dist 目录提交到 build-dist 分支
5. 每次都会清空旧文件，只保留最新打包内容
